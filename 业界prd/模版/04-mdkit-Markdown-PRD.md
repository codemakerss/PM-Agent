# mdkit Markdown PRD 模板

来源：https://mdkit.io/templates/product-requirement-document

## 适用场景

适合轻量级项目、开发者协作、Git 管理和 AI 生成。

## 模板结构

- PRD 标题、状态、负责人
- Problem Statement：为什么要做、为谁解决
- User Stories：用户故事
- Functional Requirements：功能需求清单
- Success Metrics：成功指标

## BA 使用要点

- 保持短小，适合 MVP 或单功能需求。
- 可以直接作为 BA-Agent 输出 Markdown 的最小模板。
- 缺少风险、依赖和验收标准时需要补充。
